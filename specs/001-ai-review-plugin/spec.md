# Feature Specification: AI Review WordPress Plugin

**Feature Branch**: `001-ai-review-plugin`
**Created**: 2026-03-14
**Status**: Draft
**Input**: WordPress 記事編集画面の右サイドバーに表示され、AIによる記事修正を実行できるプラグイン。プロンプト入力欄・実行ボタン・上書き注意書きを備え、設定画面でLLMプロバイダ・モデル・システムプロンプトを管理する。

## User Scenarios & Testing *(mandatory)*

### User Story 1 - AIによる記事修正 (Priority: P1)

記事の執筆者として、投稿編集画面の右サイドバーからプロンプトを入力し「実行」ボタンを押すと、AIが記事の内容を修正し、現在の記事内容を上書きする。

**Why this priority**: プラグインの核となる機能であり、これがなければプラグインの存在意義がない。

**Independent Test**: 投稿編集画面を開き、サイドバーにプロンプトを入力して実行ボタンを押し、記事内容がAIにより修正されることを確認する。

**Acceptance Scenarios**:

1. **Given** 投稿編集画面を開いている状態, **When** サイドバーのプロンプト欄に「もっとカジュアルな文体にして」と入力し実行ボタンを押す, **Then** AIが記事全体を修正し、エディタの内容が更新される
2. **Given** 投稿編集画面を開いている状態, **When** プロンプト欄が空の状態で確認する, **Then** 実行ボタンが無効化されている
3. **Given** AI処理中の状態, **When** 処理が実行されている間, **Then** ローディング表示が表示され、実行ボタンが無効化される
4. **Given** 投稿編集画面のサイドバーが表示されている状態, **When** サイドバーを確認する, **Then** 記事が上書きされる旨の注意書きが目立つ形で常時表示されている

---

### User Story 2 - LLM設定の管理 (Priority: P2)

プラグイン管理者として、WordPress管理画面の設定ページからLLMのプロバイダ、モデル名、APIキー、システムプロンプトを設定できる。

**Why this priority**: AIとの連携に必須の設定であり、これがなければAI機能が動作しない。

**Independent Test**: WordPress管理画面の設定ページを開き、プロバイダ・モデル・APIキー・システムプロンプトを入力して保存し、設定が反映されることを確認する。

**Acceptance Scenarios**:

1. **Given** プラグイン設定画面を開いた状態, **When** LLMプロバイダ名・モデル名・APIキー・システムプロンプトを入力し保存する, **Then** 設定が保存され、次回表示時にも値が保持されている
2. **Given** 設定が保存された状態, **When** 投稿編集画面でAI修正を実行する, **Then** 保存されたプロバイダ・モデル・システムプロンプトが使用される
3. **Given** プラグイン設定画面を開いた状態, **When** 必須項目（プロバイダ・モデル・APIキー）が未入力のまま保存しようとする, **Then** バリデーションエラーが表示される

---

### User Story 3 - 未設定時のガイダンス (Priority: P3)

記事の執筆者として、LLM設定が未完了の場合にサイドバーパネルで設定が必要である旨の案内が表示される。

**Why this priority**: ユーザーが設定不備に気づかず混乱することを防ぐ補助機能。

**Independent Test**: LLM設定を未入力の状態で投稿編集画面を開き、サイドバーに設定案内が表示されることを確認する。

**Acceptance Scenarios**:

1. **Given** LLM設定が未完了の状態, **When** 投稿編集画面のサイドバーを開く, **Then** 「設定画面からLLMの設定を完了してください」という案内メッセージが表示され、実行ボタンが無効化されている

---

### Edge Cases

- プロンプトが空の状態で実行ボタンを押した場合 → 実行ボタンが無効化されておりAPI呼び出しを行わない
- LLM設定が未完了（APIキーやプロバイダが未設定）の状態で実行ボタンを押した場合 → 設定が必要である旨のメッセージを表示する
- API呼び出しがタイムアウトまたはエラーになった場合 → エラーメッセージを表示し、記事内容は変更しない
- 非常に長い記事を処理する場合 → モデルのトークン上限を超える場合はエラーメッセージを表示する
- 投稿編集権限を持たないユーザーの場合 → サイドバーパネル自体が表示されない

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: プラグインは、WordPress投稿編集画面（Gutenbergエディタ）の右サイドバーにパネルとして表示されなければならない
- **FR-002**: サイドバーパネルには、プロンプト入力欄（テキストエリア）が含まれなければならない
- **FR-003**: サイドバーパネルには、AI修正を実行するための実行ボタンが含まれなければならない
- **FR-004**: サイドバーパネルには、「AIによる修正は現在の記事内容を上書きします」という注意書きが常時表示されなければならない
- **FR-005**: 実行ボタンを押すと、現在の記事内容とユーザーのプロンプトをLLMに送信し、返却された修正内容でエディタの記事内容を上書きしなければならない
- **FR-006**: AI処理中はローディング状態を表示し、実行ボタンを無効化しなければならない
- **FR-007**: API呼び出しが失敗した場合、記事内容を変更せず、ユーザーにエラーメッセージを表示しなければならない
- **FR-008**: プロンプトが空の場合、実行ボタンを無効化しなければならない
- **FR-009**: WordPress管理画面にプラグイン設定ページを追加しなければならない
- **FR-010**: 設定ページでは、LLMプロバイダ名を入力・保存できなければならない
- **FR-011**: 設定ページでは、LLMモデル名を入力・保存できなければならない
- **FR-012**: 設定ページでは、システムプロンプトをテキストエリアで入力・保存できなければならない
- **FR-013**: 設定ページでは、APIキーを入力・保存できなければならない
- **FR-014**: APIキーはフロントエンドに公開されない形で安全に保存されなければならない
- **FR-015**: 投稿編集権限を持つユーザーのみがサイドバーパネルを利用できなければならない
- **FR-016**: LLM設定が未完了の場合、サイドバーパネルに設定が必要である旨のメッセージを表示しなければならない

### Key Entities

- **プラグイン設定**: LLMプロバイダ名、モデル名、APIキー、システムプロンプトを含む設定情報
- **AIリクエスト**: ユーザーのプロンプト、現在の記事内容、システムプロンプトを含むLLMへのリクエスト
- **AI応答**: LLMから返却された修正後の記事内容

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 投稿編集画面のサイドバーからプロンプトを入力し実行ボタンを押すと、AIによる修正結果が記事に反映される
- **SC-002**: 設定画面で入力したLLMプロバイダ・モデル・システムプロンプト・APIキーが正しく保存・復元される
- **SC-003**: 上書き注意書きが、実行前に常にユーザーの目に入る位置に表示されている
- **SC-004**: API呼び出し失敗時に記事内容が保全され、ユーザーにエラーが通知される
- **SC-005**: 投稿編集権限を持たないユーザーにはAIレビュー機能が表示されない

## Assumptions

- WordPress 6.0以上のGutenberg（ブロックエディタ）環境を対象とする
- LLMプロバイダはOpenAI互換のChat Completions APIを想定する（他プロバイダも同形式であれば対応可能）
- APIキーの保存にはWordPress標準のオプションAPIを使用し、フロントエンドには公開しない
- システムプロンプトのデフォルト値として「あなたは優秀な編集者です。指示に従って記事を修正してください。修正後の記事本文のみを返してください。」を使用する
- プラグイン設定ページへのアクセスは管理者権限を持つユーザーに限定する
